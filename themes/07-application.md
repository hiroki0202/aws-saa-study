## Part 1: メッセージキューとイベント通知

アプリケーションを疎結合にするための「メッセージング」サービスを学びます。

**Amazon SQS（Simple Queue Service）**はメッセージキューサービスです。プロデューサー（送信側）がメッセージをキューに入れ、コンシューマー（受信側）がキューからメッセージを取り出して処理します。

SQSの重要な特徴:
- **プル型**: コンシューマーが能動的にキューからメッセージを取り出す
- **標準キュー**: 高スループット、順序は保証されない、少なくとも1回の配信
- **FIFOキュー**: 順序が保証される、ちょうど1回の配信（スループットは標準より低い）
- **可視性タイムアウト**: コンシューマーがメッセージを取り出した後、他のコンシューマーから一定時間見えなくなる仕組み（処理中の重複を防ぐ）
- **デッドレターキュー（DLQ）**: 処理に繰り返し失敗したメッセージを別のキューに移して分析できる

**Amazon SNS（Simple Notification Service）**はパブリッシュ/サブスクライブ型のメッセージングサービスです。トピックに対してメッセージを発行（パブリッシュ）すると、そのトピックをサブスクライブしているすべての宛先（Eメール・SQS・Lambda・HTTPエンドポイントなど）に一斉配信されます。

SQSとSNSの使い分け:
- **SQS**: 1つのキューを1つのコンシューマーが処理（非同期処理の緩衝材）
- **SNS**: 1つのメッセージを複数の宛先に同時配信（ファンアウト）

**SNS + SQS ファンアウトパターン**: SNSトピックにメッセージを発行し、複数のSQSキューをサブスクライバーとして設定することで、複数のシステムが同じイベントを並行処理できます。

## Part 2: API GatewayとStep Functions

**Amazon API Gateway**はREST API・WebSocket API・HTTP APIを構築・管理・公開するサービスです。LambdaやEC2・外部HTTP endpointへのリバースプロキシとして機能します。

API Gatewayの主な機能:
- **認証・認可**: IAM認証・Cognitoユーザープール・Lambdaオーソライザー
- **スロットリング**: リクエスト数を制限してバックエンドを保護
- **キャッシュ**: APIレスポンスのキャッシュで低レイテンシー化
- **ステージ**: dev・staging・prodなど複数の環境管理

**AWS Step Functions**はサーバーレスのワークフローオーケストレーションサービスです。複数のLambda関数や他のAWSサービスを順番に・並列に・条件分岐しながら実行するワークフローを視覚的に定義できます。単純なLambdaでは対応しにくい複雑な複数ステップ処理に使います。

## Part 3: Cognito

**Amazon Cognito**はWebアプリケーション・モバイルアプリに対するユーザー認証・認可機能を提供するサービスです。

- **ユーザープール**: ユーザーの登録・ログイン・パスワード管理を担う（サインアップ・サインイン機能）
- **IDプール（フェデレーテッドアイデンティティ）**: GoogleやFacebookなどの外部IDプロバイダーやユーザープールで認証したユーザーに、AWSリソースへのアクセス権を一時的に付与する

Cognitoは「アプリのユーザー管理とAWSリソースへの安全なアクセス」を組み合わせて実現するためのサービスです。
