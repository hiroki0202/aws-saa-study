## Part 1: S3の基本

**Amazon S3（Simple Storage Service）**は、オブジェクトストレージサービスです。ファイルシステムのように「ファイル」を管理するのではなく、「オブジェクト（データ + メタデータ）」を「バケット」に格納します。

S3の特徴:
- **耐久性**: 99.999999999%（イレブンナイン）の耐久性
- **スケーラビリティ**: 容量制限なし
- **グローバルユニーク**: バケット名はグローバルで一意

**S3ストレージクラス**はデータのアクセス頻度とコストに応じて使い分けます。

| ストレージクラス | 特徴 | 向いているケース |
|---|---|---|
| S3 Standard | 高可用性・高耐久性 | 頻繁にアクセスするデータ |
| S3 Standard-IA | アクセス頻度が低い | 月1回程度のアクセス |
| S3 One Zone-IA | 単一AZ、低コスト | 再作成可能なデータ |
| S3 Glacier Instant Retrieval | アーカイブ・即時取得 | 四半期に1回程度 |
| S3 Glacier Flexible Retrieval | アーカイブ・数分〜時間で取得 | 年1回程度 |
| S3 Glacier Deep Archive | 最安値・取得に12時間 | 規制上の保管義務 |
| S3 Intelligent-Tiering | アクセスパターン自動最適化 | アクセス頻度が不規則 |

**S3 Intelligent-Tiering**は、アクセスパターンを自動で分析し、最適なストレージクラスに自動移行します。アクセス頻度が予測できない場合に有効です。

## Part 2: S3のセキュリティと機能

S3への**アクセス制御**は複数の方法があります。

- **バケットポリシー**: バケット全体に適用するリソースベースのポリシー
- **ACL（アクセスコントロールリスト）**: 旧来の方法（現在は推奨されない）
- **IAMポリシー**: IAMユーザー・ロールに付与する権限

**S3ブロックパブリックアクセス**は、S3バケットへのパブリックアクセスをアカウントレベル・バケットレベルで一括ブロックできる機能です。誤った公開設定による情報漏洩を防ぐ重要な設定です。

**暗号化**については以下の方式があります。
- **SSE-S3**: AWSが管理するS3マネージドキーで暗号化
- **SSE-KMS**: AWS KMSのキーで暗号化（キーのログ・管理が可能）
- **SSE-C**: 顧客が提供するキーで暗号化（キー管理は自社）

**S3バージョニング**を有効にすると、オブジェクトの上書き・削除前のバージョンが保持されます。誤削除からの復旧や変更履歴の管理に使います。

**S3ライフサイクルルール**で、一定期間後に別のストレージクラスへ自動移行したり、一定期間後に自動削除したりできます。例えば「作成から30日後にStandard-IAへ移行し、1年後にGlacierへ移行し、7年後に削除」という設定が可能です。

## Part 3: EBSとEFS

**Amazon EBS（Elastic Block Store）**はEC2インスタンスにアタッチするブロックストレージです。EC2インスタンスのOSディスクやデータディスクとして使います。

重要な特性:
- **AZ内のサービス**: EBSボリュームは特定のAZに存在し、同じAZ内のEC2にのみアタッチ可能
- **単一インスタンスアタッチが基本**（io1/io2 Multiアタッチという例外もあり）
- **スナップショット**でバックアップを取得し、別のAZ・リージョンにコピー可能

**Amazon EFS（Elastic File System）**は複数のEC2インスタンスから同時にマウントできるNFSベースの共有ファイルシステムです。複数のAZにまたがって利用でき、容量は自動で増減します。コンテンツ管理・共有ストレージとして使います。

| | EBS | EFS |
|---|---|---|
| 種類 | ブロックストレージ | ファイルシステム |
| 共有 | 基本的に1インスタンスのみ | 複数インスタンスで共有可能 |
| AZ | 単一AZ内 | マルチAZ対応 |
| 容量 | 固定サイズを指定 | 自動拡張 |
