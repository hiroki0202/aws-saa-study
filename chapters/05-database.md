## Part 1: Amazon RDS

**Amazon RDS（Relational Database Service）**はリレーショナルデータベースをマネージドで提供するサービスです。MySQL・PostgreSQL・Oracle・SQL Server・MariaDB・Aurora をサポートしています。

RDSを使うメリットは、OSのパッチ適用・バックアップ・フェイルオーバーといった運用作業をAWSが担当することです。

**マルチAZ配置**はRDSの高可用性機能です。プライマリDBインスタンスがスタンバイインスタンスと同期レプリケーションを行い、プライマリに障害が起きると自動でスタンバイへフェイルオーバーします。フェイルオーバーは通常1〜2分で完了します。ただし、スタンバイは読み取りには使えません（待機専用）。

**リードレプリカ**は読み取り専用のコピーです。非同期レプリケーションで作成され、読み取りクエリをリードレプリカに向けることでプライマリの負荷を軽減できます。マルチAZ（高可用性）とリードレプリカ（スケールアウト）は目的が異なります。

| | マルチAZ | リードレプリカ |
|---|---|---|
| 目的 | 高可用性・フェイルオーバー | 読み取りスケールアウト |
| レプリケーション | 同期 | 非同期 |
| 読み取り | 不可（スタンバイ） | 可能 |

## Part 2: Amazon Aurora

**Amazon Aurora**はAWS独自開発の高性能リレーショナルデータベースです。MySQL・PostgreSQLと互換性があり、同等のオープンソースDBと比べてMySQL互換で最大5倍、PostgreSQL互換で最大3倍のスループットを発揮するとされています。

Auroraの重要な特徴:
- **ストレージ自動拡張**: 10GBから始まり、最大128TBまで自動拡張
- **6方向レプリケーション**: データを3つのAZに2コピーずつ計6コピー保存（高耐久性）
- **Aurora Serverless**: リクエスト数に応じてDBキャパシティが自動スケール（断続的なワークロードに最適）
- **Aurora Global Database**: 複数リージョンにまたがるグローバルな読み取りレプリカを持てる

**Aurora Readerエンドポイント**を使うと、複数のリードレプリカに自動で負荷分散して読み取りクエリを分散できます。

## Part 3: DynamoDB と ElastiCache

**Amazon DynamoDB**はフルマネージドのNoSQLデータベースです。キーバリュー型・ドキュメント型のデータ構造を持ちます。

DynamoDBの特徴:
- **サーバーレス**: 容量のプロビジョニングが不要
- **単一桁ミリ秒のレイテンシー**: 大量のデータでも高速
- **グローバルテーブル**: 複数リージョンにデータを自動レプリケーション
- **DAX（DynamoDB Accelerator）**: DynamoDBのインメモリキャッシュ。マイクロ秒レベルのレイテンシーを実現

**Amazon ElastiCache**はインメモリキャッシュのマネージドサービスです。頻繁にアクセスされるデータをメモリにキャッシュすることで、DBへのクエリ回数を減らし応答速度を向上させます。

- **ElastiCache for Redis**: 豊富なデータ構造・永続化・レプリケーション対応
- **ElastiCache for Memcached**: シンプルなキャッシュ、マルチスレッド対応

セッションデータ・ランキング・リアルタイム分析など、高速な読み取りが求められる場面で活用します。
